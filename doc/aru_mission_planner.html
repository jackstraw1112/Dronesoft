▌ 计算过程 · COMPUTATION
①
所需弹数 n
n = ⌈ log(1 − P̄) / log(1 − Pk) ⌉
P̄ = 0.90 → log(1−0.90) = −1.000
Pk = 0.70 → log(1−0.70) = −0.523
商 = 1.91 向上取整 → 2 弹
②
总架次 N（含备份）
N = n + 备份架数
n = 2 + 备份 = 1 = 3 架
建议出动 3架
按 2 弹齐射 + 1 架备份编组，预计达成毁伤率 ≥ 0.90

▌ 计算过程 · COMPUTATION
①
覆盖需求 N搜
N搜 = ⌈ 区域面积 / 单机搜索面积 ⌉
面积 = 78 km² ÷ 单机覆盖 = 25 km² = 4 架
▸ 单机搜索面积按武器手册取 25 km²（速度×搜索时长×扫描带宽）
②
打击需求 N打
N打 = 估计目标数 × 每目标弹数
M = 3 部 × n = 2 弹/部 = 6 架
③
综合需求 N（取大者 + 备份）
N = max(N搜, N打) + 备份
max(4, 6) = 6 + 备份 = 2 = 8 架
▸ 打击需求是瓶颈（6 > 4），区域里目标偏多
建议出动 8架
分 3 路扇形扫描区域，先到先攻，余机就近补打或转入备选区域
